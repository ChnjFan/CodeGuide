# 数据结构

Redis 对外暴露的 5 种核心数据类型（字符串、哈希、列表、集合、有序集合），底层并非单一结构，而是根据数据特征选择「基础结构 + 优化结构」的组合（比如小哈希用压缩列表，大哈希用哈希表）。

## 字符串 String

Redis 的 String 类型不使用 C 原生字符串 `char*`，而是封装了 SDS（简单动态字符串）实现。

核心特征：

- 可存储**字符串、整数、浮点数**（最大 512MB）；
- 是「二进制安全」的（可存图片、视频等二进制数据，不依赖 `\0` 终止）；
- 支持原子操作（如 `INCR`、`APPEND`）；
- 底层通过 `robj` 封装 + SDS 存储，并有 3 种编码适配不同场景。

### 底层核心结构 SDS

SDS 按照字符串长度分为不同结构体（sdshdr8/16/32/64），用最小的类型存储长度，减少内存开销：

```c
// sds.h 中 sdshdr8 结构体（最常用，存储 ≤2^8-1 字节的字符串）
struct __attribute__ ((__packed__)) sdshdr8 {
    uint8_t len;     // 已使用长度（O(1) 获取字符串长度）
    uint8_t alloc;   // 总分配长度（不含头部和终止符）
    unsigned char flags; // 标识 SDS 类型（8/16/32/64）
    char buf[];      // 变长数组：存储字符串内容（兼容 C 字符串，末尾加 \0）
};
```

SDS 通过保存已使用长度 len 来实现 O(1) 复杂度计算使用长度，提供二进制安全的类型。

| 特性       | C 字符串              | SDS                                     |
| ---------- | --------------------- | --------------------------------------- |
| 长度计算   | O (n)（遍历到 \0）    | O (1)（直接取 len 字段）                |
| 缓冲区溢出 | 无保护，易溢出        | 扩容前检查，自动分配内存                |
| 二进制安全 | 不支持（\0 视为结束） | 支持（用 len 标识结束，可存任意二进制） |
| 内存重分配 | 修改必重分配          | 预分配内存（减少重分配次数）            |

### 扩容策略

字符串扩容策略分为贪心和非贪心：

- 贪心：扩容大小小于 1 MB 时，额外扩容一倍；扩容大小超过 1 MB 时，每次多扩容 1 MB；
- 非贪心：不额外扩容，需要多少扩容多少。

SDS 扩容策略是在内存占用和扩容次数做的最优平衡。

小于 1MB 时翻倍扩容，目的是减少扩容次数，避免频繁内存的重新分配性能变差。字符串较短的时候，翻倍占用的内存非常少。

大于 1MB 后只增加 1MB 目的是防止浪费空间，持续翻倍增加很快就会占满内存。
