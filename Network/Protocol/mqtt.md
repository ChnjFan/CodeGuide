# MQTT 协议

MQTT（Message Queuing Telemetry Transport，消息队列遥测传输协议），是一种基于发布/订阅（publish/subscribe）模式的"轻量级"通讯协议，该协议构建于 TCP/IP 协议上，可以以极少的代码和有限的带宽，为连接远程设备提供实时可靠的消息服务。

## 协议原理

主流的 MQTT 协议通过 TCP 连接，UDP 版本的叫做 MQTT-SN，两种版本基于不同连接方式优缺点也不同。

### 主要特性

MQTT 协议主要工作在低带宽、不可靠的网络的远程传感器和控制设备通讯而设计的协议。

- 发布/订阅模式，提供一对多的消息发布。
- 负载内容屏蔽的消息传输。
- 使用 TCP/IP 提供网络连接。
- 三种消息发布服务质量，针对消息类型设置不同 QoS。
- 小型传输，开销小（固定头部 2 字节），降低网络流量。
- 客户端异常中断遗言机制。

### 协议实现方式

实现 MQTT 需要完成客户端与服务端之间的通信，其中消息发布者和订阅者都是客户端，消息代理是服务器。

- 发布者（Publisher）：消息发送方，只需要将消息发布到指定的主题（Topic）上，也可以是订阅者。
- 订阅者（Subscriber）：消息接收方，订阅消息 Topic 后服务端会主动推送消息。
- 服务端（Broker）：消息代理，接收发布者消息，管理 Topic 和客户端订阅关系，将消息推送给订阅者，同时还负责所有客户端的连接认证、会话管理和离线消息存储。

消息传输包含主题 Topic 和负载 payload：

- Topic：消息的类型，订阅者订阅主题后收到该主题的消息。
- payload：消息内容。
