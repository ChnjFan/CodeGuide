# C/C++ 内存模型

C/C++ 程序运行时，操作系统会为其分配**虚拟地址空间**（逻辑内存），而非直接操作物理内存 —— 这是内存模型的底层前提：

- **物理内存**：硬件层面的内存条，是实际存储数据的硬件资源；
- **虚拟内存**：操作系统为每个进程分配的 “抽象内存空间”（32 位系统通常为 4GB，64 位系统更大），通过 MMU（内存管理单元）映射到物理内存；
- **核心意义**：隔离进程内存、实现内存保护、支持内存扩容（如交换分区）。

## 虚拟地址分区

32 位系统下，进程虚拟地址空间分为五个核心区域，从低地址到高低值：

- 代码段（Text）：存储可执行指令、常量字符串等只读数据。
- 数据段（Data）：存储已初始化的全局变量和静态变量。
- 未初始化数据段（BSS）：未初始化的全局变量和静态变量，`main` 函数运行前清 0。
- 堆（Heap）：动态分配的内存，手动申请释放。
- 栈（Stack）：局部变量、函数参数、返回地址和寄存器上下文。

此外，在高地址区为内核空间，供操作系统内核使用，用户进程无法访问；在堆和栈之间的共享库/动态链接库段，用来存储共享库的代码和数据。
