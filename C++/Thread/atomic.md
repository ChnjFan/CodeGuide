# 原子操作

原子操作是不可再分、不可中断的最小执行单元。整个操作过程不可分割、不可中断，没有中间状态，要么完全执行完毕，要么完全没执行，不会被线程调度或硬件中断打断。

## 整型变量赋值

**整型变量赋值为什么不是原子操作？**

现代 CPU 架构里没有内存直连内存的硬件通路没，整型变量赋值需要借助寄存器中转才能完成内存拷贝，对应两条计算机指令：将数据从内存中读取到寄存器然后再写回内存。

```assembly
mov eax, dword ptr [b]  
mov dword ptr [a], eax
```

两条指令再多线程执行时，某个线程可能再第一条指令执行完毕后被剥夺 CPU 时间片，切换到另一个线程而产生不确定的情况。

## std::atomic

`std::atomic` 是**C++11 引入的原子模板类**，核心作用是**实现多线程环境下的原子操作**，是并发编程的基础组件。

```cpp
template<class T>
struct atomic;
```

注意：`std::atomic` 将拷贝构造删除，因此无法通过 `std::atomic<int> value = 99;` 初始化。

```cpp
std::atomic<int> value = 99; // error: use of deleted function ‘std::atomic<int>::atomic(const std::atomic<int>&)’
std::atomic<int> value_success;
value_success.store(99);
value_success = 99;
```

常用核心操作：

- `store(val)`：原子写入值
- `load()`：原子读取值
- `fetch_add(n)`：原子加法（返回旧值）
- `compare_exchange_weak`：CAS（比较并交换，无锁编程核心），相比 `compare_exchange_strong` 可能会出现伪失败（由于 CAS 操作被硬件打断，值可以修改但是返回失败）。

### 内存序

`std::atomic` 通过控制**指令重排**，保证多线程可见性和执行顺序。

编译器和 CPU 为了性能可能会重排代码指令顺序，多线程下重排可能会导致逻辑错误，无锁变成完全依赖指令执行顺序的正确性，**内存序**用来约束重排规则。

- `memory_order_seq_cst`：最严格的执行顺序，通常用在简单、调试场景。
- `memory_order_acquire`：该语句之后的代码不会出现在前面，用于读共享变量和无锁队列出队操作。
- `memory_order_release`：该语句之前的代码不会出现在后面，用于写共享变量和无锁队列入队操作。
- `memory_order_acq_rel`：前后代码顺序不会乱，用于 CAS 操作。

## CAS

CAS(Conpare And Swap) 是一条**无锁、原子指令**，CPU 硬件支持用于实现并发的变量安全修改，是**乐观锁**的核心实现。

```
CAS(内存地址 V，预期旧值 A，新值 B)
```

比较内存地址 V 的值是否等于预期旧值 A，如果相等说明期间没有其他线程修改，直接将 V 的值修改为 B，否则数据被篡改不可修改，返回失败。

### ABA 问题

线程 A 读取值后，线程 B 将值修改后又修改回来，之后线程 A 执行 CAS 发现值没有变化正常修改，实际数据中间被改动过，在逻辑上已经不一致。

无锁队列的典型场景：

1. 线程 T1 读取队列节点指针地址为 A 准备执行 CAS 修改；
2. 线程 T2 并发将地址 A 的节点从链表摘除，复用该节点地址修改内容后重新挂链；
3. 线程 T1 执行 CAS 操作判定地址未改变后直接修改，导致链表结构错误。

**解决方案**：`Boost::lockfree:queue` 无锁队列通过在 `atomic` 变更额外设置版本号 `tag`，每次进行 CAS 操作后累加版本号，通过比较版本号和值判断数据是否被其他线程修改。

### 应用场景

#### 无锁原子操作

多线程高频读写共享变量（计数器、状态标记、序号自增），使用互斥锁带来上下文切换开销。

通过 `std::atomic` 定义共享变量。

#### 自旋锁

自旋锁（SpinLock）是一种轻量级、用户态、忙等待的锁。线程抢锁失败时不阻塞、不放弃 CPU，而是循环不断重试抢锁直到获取锁。

相比互斥锁在获取锁失败后会将线程休眠阻塞，自旋锁特点：

- 不阻塞线程，无线程切换和内核调度的开销。
- 用户态实现，无内核态切换、无阻塞开销。
- 不适合长期持有锁，CPU 浪费开销。

**适用场景**：临界区代码极短，锁竞争不激烈，对延迟极其敏感的内核、高性能组件、高并发中间件和网络框架。

#### 无锁队列

无锁队列依靠 CAS 原子操作实现多线程安全的入队、出队操作，实现高并发、低延迟的队列操作。

##### 入队操作原理

1. 新建入队节点 `node`；
2. 循环 CAS 获取尾指针：
    1. 读取到尾指针 `tail` 和 `next = tail->next`；
    2. 再次读取尾指针 `tail` 进行一致性校验；
    3. 如果 `next` 说明是真实的尾指针，如果不是（被其他线程更新）修正真正的尾指针 `tail_.store(tail->node)` 后重试；
    4. CAS 操作设置  `tail->next.store(node)`，失败重试；修改尾指针为新入队节点 `tail_.store(node)`，失败重试。

##### 出队操作原理

1. 循环 CAS 获取头指针
    1. 读取到当前头指针 `head`；
    2. 读取 `head->next` 指向的节点为 `next`，如果为空说明队列为空，直接返回；
    3. CAS 操作修改头指针指向下一个节点 `head_.store(next)`，失败重试；
2. 从 `next` 中读取数据，并返回出队成功。
