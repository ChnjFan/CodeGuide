## 线程局部存储区

线程局部存储区（Thread Local Storage，简称 TLS）是**操作系统 / 编程语言为每个线程单独分配的专属内存区域**，核心特性是**线程私有、完全隔离**。

1. **隔离性**：全局变量是所有线程共享的，会引发并发安全问题；而 TLS 中存储的数据，**仅当前线程能访问**，每个线程都有自己独立的副本，线程间互不干扰。
2. **用途**：专门存放**线程专属的状态 / 上下文数据**（如线程 ID、数据库连接、日志上下文、用户会话信息），无需加锁就能保证线程安全。
3. **底层支持**：操作系统和主流编程语言都原生实现（Java `ThreadLocal`、C++ `thread_local`、C `__thread`），自动为每个线程分配 / 回收独立存储。

⚠️注意：系统的线程局部存储区域内存空间并不大，所以尽量不要利用这个空间存储大的数据块，如果不得不使用大的数据块，可以将大的数据块存储在堆内存中，再将该堆内存的地址指针存储在线程局部存储区域。

## std::thread_local

`thread_local` 是 C++11 引入的**存储期说明符**，核心作用是**让变量成为线程局部变量**——**每个线程都拥有该变量的独立副本**，线程之间互不干扰、互不共享。

1. **生命周期**：变量随**线程启动**创建，随**线程结束**销毁，和线程绑定。
2. **作用域**：可用于全局变量、静态局部变量、类的静态成员变量。
3. **隔离性**：线程 A 修改自己的副本，完全不会影响线程 B 的副本。
4. **默认初始化**：每个线程的变量都会被独立初始化。

```cpp
#include <iostream>
#include <thread>

// 线程局部全局变量
thread_local int num = 0;

void func() {
    num++;  // 每个线程修改自己的 num 副本
    std::cout << "线程ID:" << std::this_thread::get_id() 
              << " num值:" << num << std::endl;
}

int main() {
    std::thread t1(func);
    std::thread t2(func);
    t1.join();
    t2.join();
    // 输出：两个线程的 num 都是 1，互不影响
}
```

解决**多线程并发访问共享变量**的线程安全问题，无需加锁（如 mutex），天然无竞争，常用于：

- 线程专属的计数器、缓存；
- 线程独立的日志、上下文数据；
- 避免全局静态变量在多线程下的竞争问题。
